# 服务器

服务器有两层含义：

  1. 软件层面：软件，可以把本地资源发布到网络中，提供给网络中其他用户访问，比如Tomcat。在软件层面，服务器是对外处理请求、提供服务的软件
  2. 硬件层面：一台性能比较高的计算机，为很多用户提供服务。

服务器开发：指的是在服务器软件程序中，进一步去编写程序去运行。

# 手写简易服务器

服务端程序

package com.demo.MyServer;

import java.io.*;
import java.net.ServerSocket;
import java.net.Socket;
import java.util.Arrays;

public class Server {
    public static void main(String[] args) throws IOException {
        //创建服务端套接字，监听端口为8080
        ServerSocket serverSocket = new ServerSocket(8080);
        while(true) {
            //接收来自于客户端的请求，建立连接，是一个阻塞方法
            Socket accept = serverSocket.accept();
            //从Socket中获取输入流
            InputStream in = accept.getInputStream();
            //byte数组用于存储读取的数据
            byte[] requestData = new byte[65535];
            //读取数据
            int len = in.read(requestData);
            //获取有效数据
            String s = new String(requestData, 0, len);
            //对获取到的数据进行处理
            ServerProcess p = new ServerProcess();
            p.process(s);
            //获取客户请求的资源名称
            String resource = p.getResource().substring(1);
            System.out.println(resource);
            File file = new File(resource);
            //获取Socket的输出流
            OutputStream outputStream = accept.getOutputStream();
            System.out.println(ClassLoader.getSystemResource("").getPath());

            if(file.exists()) {
                //如果该文件存在，就执行下面的内容
                System.out.println("200");
                //读取请求的内容
                InputStream inputStream = new FileInputStream(file);
                byte[] responseData = new byte[65535];
                int readCount = inputStream.read(responseData);
                byte[] sendData = Arrays.copyOf(responseData, readCount);
                StringBuffer stringBuffer = new StringBuffer();
                stringBuffer.append("HTTP/1.1 200 OK\r\n");
                stringBuffer.append("Server: LangGan\r\n");
                stringBuffer.append("Content_type: html\r\n\r\n");
                
                outputStream.write(stringBuffer.toString().getBytes());
                outputStream.write(sendData);
                outputStream.close();
            } else {
                //如果该文件不存在，就执行下面的内容
                System.out.println("404");
                StringBuffer stringBuffer = new StringBuffer();
                stringBuffer.append("HTTP/1.1 404 File Not Found\r\n");
                stringBuffer.append("Server: LangGan\r\n");
                stringBuffer.append("Content_type: html\r\n\r\n");
                stringBuffer.append("<div>404 File Not Found</div>");

                outputStream.write(stringBuffer.toString().getBytes());
            }
        }
    }
}

加工处理请求报文的类

package com.demo.MyServer;

import java.util.HashMap;

public class ServerProcess {
    //请求方法
    private String method;
    //请求的资源名称
    private String resource;
    //协议版本
    private String protocol;
    //用一个Map存储header的信息
    private HashMap<String, String> header;

    public String getMethod() {
        return method;
    }

    public String getResource() {
        return resource;
    }

    public String getProtocol() {
        return protocol;
    }

    public HashMap<String, String> getHeader() {
        return header;
    }

    //处理请求报文
    public void process(String s) {
        int index1 = s.indexOf("\r\n");
        int index2 = s.indexOf("\r\n\r\n");
        String firstLineStr = s.substring(0, index1);
        String headerStr = s.substring(index1 + 2, index2);
        processFirstLine(firstLineStr);
        processHeader(headerStr);
    }

    //获取请求的请求方法、请求资源名称、协议版本
    private void processFirstLine(String firstLineStr) {
        String[] s = firstLineStr.split(" ");
        method = s[0];
        resource = s[1];
        protocol = s[2];
    }

    //处理请求报文的header
    private void processHeader(String headerStr) {
        header = new HashMap<>();
        String[] split = headerStr.split("\r\n");
        for (String s : split) {
            String[] keyValue = s.split(": ");
            header.put(keyValue[0], keyValue[1]);
        }
    }
}

# 部署资源

# 直接部署

直接将资源文件放置在webapp目录下。

访问方式：http://localhost:8080/文件夹名称/相对路径

# 虚拟映射

# conf/Catalina/localhost

在conf/Catalina/localhost目录下创建一个xml文件，在xml文件中配置Context节点信息。

比如现在我们创建了一个user.xml，在docBase属性中填写资源的地址，然后通过

<?xml version="1.0" encoding="UTF-8"?>
<Context docBase="D:\app1"/>

docBase是要访问的文件的实际路径。

访问方式：http://localhost:8080/{xml的名称}/

# conf/server.xml

需要在节点（标签）内配置Context

<Context path="/app" docBase="D:\webapp"/>

docBase是要访问的文件的实际路径，path是用于访问自定义的名字。

访问方式：http://localhost:8080/{path的值}/

# 缺省端口、缺省应用、缺省页面

# 缺省端口

如果想让一个访问不用携带端口号，那么服务器就要使用当前协议默认的端口号

http: 80

https: 443

在conf -> server.xml中可以配置端口号

<Connector port="80" protocol="HTTP/1.1" connectionTimeout="2000" redirectPort="8443"/>

# 缺省应用

在没有应用名称时访问缺省应用（默认应用）。

缺省应用在访问时不用携带应用名，Tomcat没有找到匹配的应用时，则将该请求交给缺省应用来处理。

  1. （直接部署）在webapp目录下创建一个ROOT目录
  2. （虚拟映射）在conf/Catalina/localhost目录下创建一个ROOT.xml
<?xml version="1.0" encoding="UTF-8"?>
<Context docBase="D:\app1"/>

缺省应用配置中，虚拟映射优先级高于直接部署。

# 缺省页面

比如访问http://localhost

没有应用名和要访问的文件名称，则访问的是缺省应用的缺省页面（默认页面）。

缺省页面（默认页面）在conf/web.xml文件中配置

<welcome-file-list>
        <welcome-file>index.html</welcome-file>
        <welcome-file>index.htm</welcome-file>
        <welcome-file>index.jsp</welcome-file>
</welcome-file-list>
