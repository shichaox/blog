# 服务注册中心

# 什么是服务注册中心、服务的注册？

服务注册中心：用于管理服务的状态的信息(ip地址、端口号等)

服务的注册：在微服务中，一个服务要调用另外一个服务需要通过http请求调用。而怎么获取这个服务的ip地址和端口号呢？

一个服务在配置注册中心客户端后，会将自己的ip地址、端口号等信息注册到注册中心。

其他配置了注册中心客户端的服务在运行时会从注册中心获取注册表信息，它就可以知道其他服务的地址等信息，就可以通过服务名获取对应的地址和端口信息从而调用该服务。

对于每个服务

  • 在服务启动时，会将自己的ip地址、端口号等信息注册到注册中心。即实现服务的注册。
  • 在服务运行的过程中，会实时向注册中心报告自己的状态
  • 在服务启动时，该服务会去服务中心拉取其他服务的信息
  • 在服务运行期间，服务会从注册中心拉取最新的服务注册信息

④ 通过DiscoveryClient对象根据服务名称获取服务实例列表

导包错误：

import com.netflix.discovery.DiscoveryClient;

应该导:

import org.springframework.cloud.client.discovery.DiscoveryClient;

⑤ 通过服务实例可以获取服务实例的URI

使用：

    @Autowired
    OrderMapper orderMapper;

    @Autowired
    RestTemplate restTemplate;

    @Autowired
    DiscoveryClient discoveryClient;

	public Order queryOrderById(Integer id) {
        // 1. 查询订单
        Order order = orderMapper.selectOrderById(id);

        // 2. 调用user-service服务获取地址
        //④ 通过DiscoveryClient对象根据服务名称获取服务实例列表
        List<ServiceInstance> instances = discoveryClient.getInstances("user-service");
        ServiceInstance serviceInstance = instances.get(0);
        //⑤ 通过服务实例可以获取服务实例的URI
        String url = serviceInstance.getUri() + "/user/address/{id}";
        String address = restTemplate.getForObject(url, String.class, order.getUserId());
        // 3. 为order对象设置地址信息
        order.setAddress(address);
        return order;
    }

# 指定Ribbon负载均衡策略

策略 实现类 描述
随机策略 RandomRule 随机选择server
轮训策略 RoundRobinRule 轮询选择
重试策略 RetryRule 对选定的负载均衡策略(轮训)之上的重试机制，在一个配置时间段内当选择服务不成功，则一直尝试使用该策略选择一个可用的服务；
最低并发策略 BestAvailableRule 逐个考察服务，如果服务断路器打开，则忽略，再选择其中并发连接最低的服务
可用过滤策略 AvailabilityFilteringRule 过滤掉因一直失败并被标记为circuit tripped的服务，过滤掉那些高并发链接的服务（active connections超过配置的阈值)
响应时间加权重策略 WeightedResponseTimeRule 根据server的响应时间分配权重，响应时间越长，权重越低，被选择到的概率也就越低。响应时间越短，权重越高，被选中的概率越高，这个策略很贴切，综合了各种因素，比如：网络，磁盘，io等，都直接影响响应时间
区域权重策略 ZoneAvoidanceRule 综合判断服务所在区域的性能，和服务的压力，轮询选择server并且判断一个AWS Zone的运行性能是否可用，剔除不可用的Zone中的所有server

指定方式有两种

# 配置文件
# 这里的users是我们的服务名称(应用名称)
users:
	ribbon:
	# 这一行配置的就是实现具体负载均衡策略实现类的全类名
		NFLoadBalancerRuleClassName: com.netflix.loadbalancer.RandomRule
# 配置类

① 定义配置类

@Configuration
public class RibbonConfig {

    @Bean
    public IRule MyRibbonRule() {
        return new RoundRobinRule();
    }
}

② 定义Ribbon客户端配置

@Configuration
@RibbonClient(name = "user-service", configuration = RibbonConfig.class)
public class MyRibbonConfig {
}

③ 一定注意，我们自己定义的配置类不能被ComponentScan扫描到，因为如果被扫描到的话，所有服务调用的负载均衡都是使用同一个我们所定义的这个。（启动类上面添加@Component注解，将自定义的Ribbon配置类不要写入扫描目录就行了）

# 详细步骤

① 添加依赖

<dependency>
	<groupId>org.springframework.cloud</groupId>
	<artifactId>spring-cloud-starter-openfeign</artifactId>
</dependency>

② 定义用来调用用户服务的OpenFeign接口

在该接口中将你需要调用的服务的方法“原封不动”的复制到接口中，方法名可以不同。

@FeignClient("user-service")
public interface UserFeignClient {

    @RequestMapping("/user/address/{id}")
    public String address(@PathVariable("id") Integer id);
}

③ 在启动类上面添加注解@EnableFeignClients

④ 调用服务，使用上面定义的接口，通过调用上面接口的方法完成服务调用。

上面的接口通过@autowired自动注入

	@Autowired
    OrderMapper orderMapper;

    @Autowired
    UserFeignClient userFeignClient;
	public Order queryOrderById(Integer id) {
		// 1. 查询订单
        Order order = orderMapper.selectOrderById(id);
        // 2. 利用openFeign发起http请求，调用user-service中的方法，查询地址
        String address = userFeignClient.address(id);
        // 3. 为order对象设置地址信息
        order.setAddress(address);
        return order;
    }

# Profile配置

每个服务自己的配置

① 每个服务自己的配置在nacos配置中心配置集的id(即data-id)满足下面的格式：（在nacos服务端网页“配置管理”中配置）

${prefix}-${spring.profiles.active}.${file-extension}
  • prefix默认为spring.application.name的值（应用名称）
  • spring.profiles.active为当前环境对应的profile（
  • file-extension为配置内容的数据格式。目前只支持properties和yml

举例：

应用名称为：user-service

环境为：dev

文件格式：yaml

user-service-dev.yaml

② 添加nacos配置中心依赖

<dependency>
	<groupId>com.alibaba.cloud</groupId>
	<artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
</dependency>

③ 在resources目录下添加bootstrap.yml配置文件

spring:
  application:
    name: user-service
  profiles:
    active: dev
  cloud:
    nacos:
      config:
        file-extension: yaml
        server-addr: localhost:8848
        #group: DEV_GROUP
        #namespace: b6d9c7c0-8189-4050-a83f-8bff345c0157
        # 可以指定多个共享配置的配置集
        shared-configs:
          - data-id: common.yaml
            group: SHARED_GROUP
            refresh: true
# ${prefix}-${spring.profiles.active}.${file-extension}
# user-service-dev.yaml
# user-service-test.yaml

# Nacos配置的持久化

将数据库改为MySql数据库

① 在数据库中创建一个名为nacos的数据库

② 在nacos数据库中执行数据库初始化文件(该文件在nacos的conf目录下)

③ 修改conf/application.properties文件，增加mysql数据源的设置，设置数据源的url、用户名、密码等信息。

db.password.0=youdontknow
