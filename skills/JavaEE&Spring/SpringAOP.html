# Spring AOP

# AOP

AOP(Aspect Oriented Programming)：面向切面编程

背后的思想是动态代理，代理可以在原有的核心业务的基础上添加额外的功能，这样就可以将比较常用的非核心业务提取出来，由代理类替我们完成

面向切面编程：定义一个切面类，切面里面定义了切入点(需要使用代理的地方)和通知方法(代理增加的额外的方法)，在我们使用切入点范围内的类的对象时，Spring会在指定的位置自动添加上额外的方法。

# AspectJ使用方法

使用步骤：

  1. 引入AspectJ依赖

  2. 在配置类上写注解@EnableAspectJAutoProxy

  3. 写一个类用于定义Aspect， 在这个类上面写注解@Component和@Aspect

  4. 在这个Aspect类里面写pointCut切入点表达式

    切入点表达式有三种形式：

    ① execution()

    ② @annotation()

    ③ @target()

  5. 写通知方法，在方法上写注解下列注解，不同的注解决定通知方法相对委托类方法执行的位置

详细步骤：

  1. 引入AspectJ依赖

    <dependency>
    <groupId>org.aspectj</groupId>
    <artifactId>aspectjweaver</artifactId>
    <version>1.9.6</version>
</dependency>

  2. 在配置类上写注解@EnableAspectJAutoProxy

    @Configuration
@ComponentScan("com.demo.aop")
@EnableAspectJAutoProxy
public class ApplicationConfig {
}

  3. 写一个类用于定义Aspect， 在这个类上面写注解@Component和@Aspect

    @Component
@Aspect
public class LogAspect {
    @Pointcut("@annotation(com.demo.aop.annotation.LogFlag)")
    public void logPointCut() {

    }

    @Before("logPointCut()")
    public void log(JoinPoint joinPoint) {
        String methodName = joinPoint.getSignature().getName();
        String className = joinPoint.getTarget().getClass().getName();
        LogData logdata = LogData.builder()
                .operation(className + "." + methodName)
                .addTime(new Date())
                .success(true)
                .build();
        System.out.println(logdata);
        SqlSession sqlSession = MyBatisUtil.getSqlSession();
        LogMapper mapper = sqlSession.getMapper(LogMapper.class);
        int affectedRow = mapper.insert(logdata);
        sqlSession.commit();
        sqlSession.close();
    }
}

  4. 在这个Aspect类里面写pointCut切入点表达式

    什么是切入点？

    切入点：是定位连接点的方式，是用于确定需要增强的范围

    连接点：可以使用通知的地方。

    增强：执行额外的方法。

    切入点表达式有三种形式：

    ① execution()

    execution(修饰符 返回值 包名.类名.方法名(参数列表))
    修饰符 可以省略不写
    返回值 可以使用通配符"*"
    包名、类名、方法名、形参 一个任意值"*"，多个任意值".."

    ② @annotation()

    @annotation(注解的全限定类名)

    注解写在组件的哪个方法上，哪个方法就划入切入点的范围

        @Pointcut("@annotation(com.demo.aop.annotation.LogFlag)")
    public void logPointCut() {
    }

    ③ @target()

    @target(注解的全限定类名)

    注解写在哪个组件上，哪个组件的方法就划入切入点的范围

  5. 写通知方法，在方法上写下列注解，不同的注解决定通知方法相对委托类方法执行的位置

    注解 解释
    @Before 在委托类方法之前执行
    @AfterReturning 在委托类有返回值之后执行
    @AfterThrowing 在委托类抛异常后执行
    @After 在委托类方法之后执行
    @Around 环绕，委托类在该方法中执行

    通知方法：在委托类基础上额外执行的方法。

    委托类：面相切面编程主要思想就是动态代理，在委托类的基础上做额外的事，委托类就是被代理的类。
