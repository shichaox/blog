# 单例模式

# 饿汉模式

在类加载时就创建了实例对象，也就是会说，该实例对象在使用前就创建了

public class Singleton1 {
    public String type = "饿汉模式";
    private static final Singleton1 instance = new Singleton1();

    //构造器私有化
    private Singleton1() {
    }

    public static Singleton1 getInstance() {
        return instance;
    }
}

# 懒汉模式(线程不安全)

当第一次使用到这个实例对象的时候，才会创建这个实例。

本处例子是线程不安全的，

public class Singleton2 {
    public String type = "饿汉模式(线程不安全)";
    private static Singleton2 instance;

    //构造器私有化
    private Singleton2() {}

    public static Singleton2 getInstance() {
        //如果instance为空就说明还没有创建过对象实例
        if(instance == null) {
            //创建instance
            instance =  new Singleton2();
        }
        return instance;
    }
}

# 懒汉模式(线程安全)

当第一次使用到该对象实例的时候，才会创建该对象实例。

本处例子是线程安全的，但是synchronized关键字写在getInstance方法上会降低该方法的并发量。无论该对象实例有没有被创建，所有调用都要争抢锁

public class Singleton3 {
    public String type = "懒汉模式(线程安全)";
    private static Singleton3 instance;

    //私有化构造器
    private Singleton3() {}

    //使用synchronized关键字，保证线程安全
    public static synchronized Singleton3 getInstance() {
        if(instance == null) {
            instance = new Singleton3();
        }
        return instance;
    }
}

# 懒汉模式(双重检查)

当第一次使用到该对象实例的时候，才会创建该对象实例。

本处例子是线程安全的，对上一种方案进行了优化。

synchronized关键字不是修饰整个方法，而是修饰创建实例的代码块，可以提高并发。只有当对象实例没有被创建时，才会去争抢锁，当对象实例已经被创建过了，就不会去争抢锁了，就会直接返回该对象实例。

public class Singleton4 {
    public String type = "懒汉模式(双重检查)";
    private static Singleton4 instance;

    private Singleton4() {}

    public static Singleton4 getInstance() {
        if(instance == null) {
            //为什么有第二次判空？因为在此处可能有其他程序已经创建了该对象。
            synchronized (Singleton4.class) {
                if(instance == null) {
                    instance = new Singleton4();
                }
            }
        }
        return instance;
    }
}

# 静态内部类

public class Singleton5 {
    public String type = "懒汉模式(静态内部类实现)";

    private Singleton5() {}

    private static class SingletonHolder{
        private static Singleton5 instance = new Singleton5();
    }

    public static Singleton5 getInstance() {
        return SingletonHolder.instance;
    }
}

# 工厂方法模式

@Data
public class Car {
    String name;

    public Car(String name) {
        this.name = name;
    }
}

public interface CarFactory {

    public Car getCar();
}

public class BydCar extends Car{

    public BydCar(String name) {
        super(name);
    }
}

public class BydCarFactory implements CarFactory{

    @Override
    public Car getCar() {
        return new BydCar("比亚迪");
    }
}

public class JiliCar extends Car{

    public JiliCar(String name) {
        super(name);
    }
}

public class JiliCarFactory implements CarFactory {

    @Override
    public Car getCar() {
        return new JiliCar("吉利汽车");
    }
}

测试

@Test
    public void factoryMethodTest() {
        CarFactory bydCarFactory = new BydCarFactory();
        CarFactory jiliCarFactory = new JiliCarFactory();

        Car bydCar = bydCarFactory.getCar();
        Car jiliCar = jiliCarFactory.getCar();

        System.out.println("bydCar = " + bydCar);
        System.out.println("jiliCar = " + jiliCar);
    }

运行结果

bydCar = Car(name=比亚迪)
jiliCar = Car(name=吉利汽车)

# 建造者模式

需要创建实例对象的类Car

@Data
public class Car {
    String name;
    String brand;
    Integer seat;
    BigDecimal price;
    String productionDate;
}

用于创建Car的实例对象的类CarBuilder

public class CarBuilder {
    private Car car = new Car();

    public CarBuilder name(String name) {
        this.car.setName(name);
        return this;
    }

    public CarBuilder brand(String brand) {
        this.car.setBrand(brand);
        return this;
    }

    public CarBuilder seat(Integer seat) {
        this.car.setSeat(seat);
        return this;
    }

    public CarBuilder price(BigDecimal price) {
        this.car.setPrice(price);
        return this;
    }

    public CarBuilder productionDate(String productionDate) {
        this.car.setProductionDate(productionDate);
        return this;
    }

    public Car build() {
        return this.car;
    }
}

测试

public class CarBuilderTest {
    public static void main(String[] args) {
        Car car = new CarBuilder()
                .name("红旗---")
                .brand("一汽红旗")
                .seat(5)
                .price(new BigDecimal("200000"))
                .productionDate("2022-08-20")
                .build();
        System.out.println(car);
    }
}

输出：

Car(name=红旗---, brand=一汽红旗, seat=5, price=200000, productionDate=2022-08-20)

工厂模式：忽略创建对象的细节，提供参数即可

建造者模式：可以由用户设置具体的参数

# 通过反射和invokeHandler实现代理

委托类是通过代理想要调用的类

代理类和委托类

  1. 方法名相同
  2. 参数相同
  3. 返回值类型相同
  4. 代理类里面调用了委托类的方法

代理类中完成的事

  1. 代理类要实现委托类对应的接口
  2. 为需要调用的委托类的方法在代理类中创建静态成员变量
  3. 通过反射获取委托类的方法
  4. 为invokeHandler创建成员变量
  5. 通过构造方法给invokeHandler赋值
  6. 在代理类中的每个方法中调用InvokeHandler.invoke()方法，在invokeHandler.invoke()方法中通过反射调用委托类对应的方法以及完成其他额外的任务。

使用invokeHandler的好处是可以通过创建不同的InvokeHandler接口的匿名子类指定不同的额外的任务

public void invoke(Method method, Object[] args) {
    //代理要添加的额外的任务
	System.out.println("开启事务");
    //通过反射调用委托类中的方法
	method.invoke(new UserServiceImpl(), new Object[]{});
}

InvokeHandler的作用：

  1. 通过反射执行委托类的方法
  2. 执行其他额外的功能

示例

代理类：

package com.demo.proxy;

import java.lang.reflect.InvocationTargetException;
import java.lang.reflect.Method;

public class UserServiceProxy implements UserService{

    private static Method create;
    private static Method delete;
    private static Method query;
    private static Method update;

    static {
        try {
            create = UserServiceImpl.class.getDeclaredMethod("create");
            delete = UserServiceImpl.class.getDeclaredMethod("delete");
            query = UserServiceImpl.class.getDeclaredMethod("query");
            update = UserServiceImpl.class.getDeclaredMethod("update");
        } catch (NoSuchMethodException e) {
            e.printStackTrace();
        }
    }

    InvokeHandler invokeHandler;

    public UserServiceProxy(InvokeHandler invokeHandler) {
        this.invokeHandler = invokeHandler;
    }

    @Override
    public void create() throws InvocationTargetException, IllegalAccessException {
        invokeHandler.invoke(create, new Object[]{});
    }

    @Override
    public void delete() throws InvocationTargetException, IllegalAccessException {
        invokeHandler.invoke(delete, new Object[]{});
    }

    @Override
    public void query() throws InvocationTargetException, IllegalAccessException {
        invokeHandler.invoke(query, new Object[]{});
    }

    @Override
    public void update() throws InvocationTargetException, IllegalAccessException {
        invokeHandler.invoke(update, new Object[]{});
    }
}

InvokeHandler接口：

public interface InvokeHandler {
    public void invoke(Method method, Object[] args) throws InvocationTargetException, IllegalAccessException;
}

代理使用实例：

package com.demo.proxy;

import java.lang.reflect.InvocationTargetException;
import java.lang.reflect.Method;

public class UserServiceProxyCase {
    public static void main(String[] args) throws InvocationTargetException, IllegalAccessException {
        UserServiceProxy userServiceProxy = new UserServiceProxy(new InvokeHandler() {
            @Override
            public void invoke(Method method, Object[] args) throws InvocationTargetException, IllegalAccessException {
                //代理要添加的额外的任务
                System.out.println("开启事务");
                //通过反射调用委托类的方法
                method.invoke(new UserServiceImpl(), new Object[]{});
            }
        });
        userServiceProxy.create();
        userServiceProxy.delete();
        userServiceProxy.query();
        userServiceProxy.update();
    }
}
