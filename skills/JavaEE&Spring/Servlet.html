# Servlet

Servlet的任务是什么？

chatGPT回答：

当你在互联网上浏览网页时，每当你点击链接或提交表单时，网站需要处理你的请求并返回相应的页面或数据。这就是 Servlet 的任务。从技术上讲，Servlet 是在服务器上运行的一种 Java 程序，它接收来自客户端的请求并生成响应。简单来说，Servlet 就像是一个服务员，负责接待客户的请求并提供相应的服务。

想象一下你去一家餐厅吃饭。你坐下后，服务员过来询问你的点菜需求。然后，他把你的订单交给厨师，并在食物准备好后将它们送到你的桌子上。在这个例子中，服务员就是 Servlet，他接收并处理你的点菜请求，并将相应的信息传递给厨师（服务器）来准备食物（生成响应）。最后，他将食物（响应）送到你的桌子上，你就可以享受美食了。

所以，Servlet 的任务是接收来自客户端的请求并生成相应的响应，使得网站能够为用户提供所需的信息或功能。

自己的理解：

一条url请求的处理过程就好比顾客去餐馆吃饭的流程。 顾客向服务员点餐，说我要吃啥，服务员就请求厨师做菜，厨师做菜的时候需要食材和原料，就叫帮手去仓库取材料，帮手取完材料将材料交给厨师，厨师做完菜后把菜交给服务员，服务员再把菜端到顾客的餐桌上，这就完成了一次请求处理的完整过程。

上述的例子中顾客点菜就相当于发起一个url请求，Servlet就是服务员，Service就是厨师 ，DAO层就是帮手。 在一次请求中，url向Servlet（服务员）发起请求，Servlet就调用Service层（厨师）来加工需要的东西，加工需要的原材料（也就是数据）由帮手DAO层取出，Service厨师将数据加工后转交给服务员Servlet，服务员Servlet再把菜品也就是结果给到请求者。

# Servlet的作用

首先，Servlet是运行在服务器的java代码，运行在Tomcat等服务器中用于响应请求。在使用Tomcat作为服务器时，每一个Servlet就像Tomcat中的一个小程序，Tomcat为Servlet提供了运行的平台。

当浏览器访问一个URL时，Servlet用于处理这个请求。也就是说，当浏览器有一个URL请求时，会触发该URL对应的servlet处理该请求，执行对应的Servlet代码。在Servlert中可以决定如何响应该请求，针对该请求做什么操作。

简而言之，你通过URL想要获得的结果是由Servlet完成的。

如果该URL没有对应的Servlet，则会由缺省的Servlet处理，Tomcat提供缺省的Servlet，也可以自己提供缺省Servlet，将自定义的Servlet的URL-Pattern。通过Tomcat默认的servlet只能访问静态资源（比如：文本类文件、图片...）。

一个URL由一个Servlet处理，一个Servlet可以处理一个或一类URL请求，这取决于你对该Servlet应该响应哪个URL的设置(URL-Pattern的设置)

# Servlet和Tomcat的关系

Tomcat就是一个Servlet容器，Tomcat为Servlet提供运行的环境。

示例：

UserServlet.java文件

当浏览器访问localhost:8080/{应用名}/user的时候，UserServlet.java就会执行，在控制台输出"hello"。

**你希望通过访问一个URL来完成某件事，这件事是由该URL对应的Servlet完成的，就叫该Servlet去替你完成。**Servlet就是用来完成URL的需求。

我想通过某个URL完成某个需求，就在该URL对应的Servlet java代码中写具体的实现代码。

怎么确定URL和Servlet的对应关系？在Servlet java代码的类名上写注解@WebServlet("/user")表示该Servlet对应的URL是"/user"

例如：

  1. 我希望当我访问localhost:8080/{应用名}/user时，控制台输出"hello"，你就可以在UserServlet.java中的doGet方法中执行输出语句。
  2. 我希望当我访问localhost:8080/{应用名}/goods时，服务端会返回所有商品信息，那么你就需要在UserServlet.java中执行相关SQL查询语句，然后将信息返回给前端。

package com.demo.servlet_demo.servlet;

import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet("/user")
public class UserServlet extends HttpServlet {

    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        System.out.println("hello");
    }
}

@WebServlet("/user")适用于标记该Servlet是响应哪个URL的。

package com.demo.servlet_demo.servlet;

import com.demo.servlet_demo.bean.User;
import com.demo.servlet_demo.mapper.UserMapper;
import com.demo.servlet_demo.util.MybatisUtil;
import org.apache.ibatis.session.SqlSession;

import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.util.List;

@WebServlet("/user")
public class UserServlet extends HttpServlet {

    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        SqlSession sqlSession = MybatisUtil.getSqlSession(true);
        UserMapper userMapper = sqlSession.getMapper(UserMapper.class);
        List<User> users = userMapper.queryAllUser();
        for (User user : users) {
            System.out.println(user);
        }
        System.out.println("hello");
    }
}
