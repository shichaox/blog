# Request & Response

Request和Response是Servlet提供的两个对象分别对应请求报文和响应报文

Servlet将请求报文转换成Request对象，将Response对象转换成响应报文。

我们直接把Request对象和Response对象拿来直接用就行了。

# Request

# 请求行

信息 内容 方法 返回值
请求方法 POST、GET getMethod() String
URL http://localhost:8080/admin/auth/login getRequestURL() StringBuffer
URI /admin/auth/login getRequestURI() String
context-path 获得上下文 /admin getContextPath() String
服务器ip getLocalAddr() String
端口号 getLocalPort() int
QueryString URL携带的参数（GET请求携带的参数） getQueryString String
协议 通讯协议版本号（例：HTTP/1.1） getProtocol() String

# 请求头

方法 返回值 说明
getHeaderNames() Enumeration 获得所有的请求头，可以通过遍历的方式来使用，使用方式类似于iterator
getHeader(String) String 传入请求头的key，返回请求头的value

# 请求体

方法 返回值 描述
getInputStream() ServletInputStream 字节流
getReader() BufferReader 字符流

# 请求参数

  1. 请求参数在请求行中

    getQueryString()

  2. 请求参数在请求体中

    getInputStream()/getReader

# 直接获取

使用getParameterXXX方法可直接获取参数

方法 返回值 说明
getParameterName() Enumeration 获取所有的key
getParameterValues(String) String[] 获得所有值
getPerameterMap() Map<String, String[]> 获取所有的请求参数
getParameter(String) String 获取第一个值

# 参数为引用类型

将参数封装成一个对象

利用反射思想，按照参数名给对应对象的属性赋值，这样这个方法就变得很通用，可以给任何的对象的属性赋值。

反射工具BeanUtils可以实现上述需求。

针对这种请求体有多中数据类型的请求，Request对象提供的的有getPart(String s)方法

getPart(String s)方法会按参数的名字获取参数对应的Part对象，每个参数对应一个Part对象。

Part对象的方法

方法 返回值 说明
getInputStream() InputStream 可以用于读取参数
getContentType() String 获取正文类型
getSize() long 获取文件字节大小
getName() String 获取请求参数名称
getSubmittedFileName String 获去上传文件的原始文件名

# 特殊的响应头

# refresh

resp.setHeader("refresh", "3;url=/demo/hello");
resp.setHeader("refresh", "1");

# Content-Type

resp.setHeader("content-type", "text/html;charset=utf-8");

# content-disposition

设置文件下载响应头，指示如何处理响应内容

一般有两种方式；

  • inline：直接在页面显示
  • attachment：以附件形式下载

"attachment;filename=1.jpg"中"1.jpg"为下载时附件的名字

resp.setHeader("content-diposition", "attachment;filename=1.jpg")

以系统时间命名下载附件文件名

resp.setHeader("content-dispostion", "attachment;filename=" + System.currentTimeMillis + ".jpg");

# location

重定向

resp.setHeader("location", "www.baidu.com");

# 问题

String contextPath = request.getServletContext().getRealPath("/");

按contextPath + "/wx/storage/fetch/" + name路径存储图片

  1. 文件路径不存在

    out = new FileOutputStream(contextPath + "/wx/storage/fetch/" + name);

  2. name本身就是完整的文件名(例：XXX.jpg)，不用加后缀名。

    File file = new File(contextPath + "/wx/storage/fetch/" + name + ".jpg");
if(!file.exists()) {
    file.mkdirs();
}

  3. 会将文件名也创建成文件夹，那么指定的路径是目录而不是文件，会拒绝访问。

    错误1

    File file = new File(contextPath + "/wx/storage/fetch/" + name);
if(!file.getParentFile().exists()) {
    file.mkdirs();
}

    错误2

    File file = new File(contextPath + "/wx/storage/fetch/" + name);
if(!file.exists()) {
    file.mkdirs();
}

    正确

    File file = new File(contextPath + "/wx/storage/fetch/" + name);
if(!file.getParentFile().exists()) {
    file.getParentFile().mkdirs();
}
