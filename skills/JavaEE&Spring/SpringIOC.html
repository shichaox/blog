# Spring IOC

IOC(Inverse of Control):存

DI(Dependency Injection):取

# 控制反转(IOC)

控制：创建对象的权利

反转：

  • 将对象的创建权利交给IOC容器
  • 将对象和对象之间关系的维护权交给IOC容器

IOC容器：生成并管理对象的抽象空间，由IOC管理的对象叫bean对象。

# 为什么要将对象的创建权利交给IOC容器？

# ① 解耦

举例：比如有10个Controller依赖了UserService，最开始实例化的是UserServiceImpl

UserService userService = new UserServiceImpl();

后面需要换一个实现类UserServiceImpl2

UserService userService = new UserServiceImpl2();

那就需要逐个修改那几个Controller

使用IOC管理对象后，只需要修改实现类就行了，无需修改使用到该实现类的代码。

# ② 复用对象

在JavaEE中，每使用某个类的对象一次就实例化一次，会造成资源的浪费，而spring的IOC思想就是将一个类的对象存入IOC容器中，等到需要使用的时候，就从容器中取即可，这样由spring管理的类(组件)，都只有唯一的对象。

组件注册：决定把哪些组件放入容器

准备工作：引入spring-context依赖

<dependencies>
    <!--引入依赖 beans、context、aop、expression、core、jcl 5+1-->
    <!--spring-context-->
    <dependency>
        <groupId>org.springframework</groupId>
        <artifactId>spring-context</artifactId>
        <version>5.2.5.RELEASE</version>
    </dependency>
    <dependency>
        <groupId>junit</groupId>
        <artifactId>junit</artifactId>
        <version>4.12</version>
        <scope>test</scope>
    </dependency>
</dependencies>

# 方式一：类直接注册

想要注册哪一个类，就在哪一个类上面加注解就行了

① 首先要打开组件扫描开关，在配置类中加@ComponentScan注解

② 在要注册的类上面加@Component注解

其他注解：

注解 解释
@Service 通常用于service层
@Repository 通常用于DAO层
@Configuration 配置类组件
@Controller

一般Component组件是通用的，不管是在哪一层都可以用。

示例：

我要将UserServiceImpl添加到容器

① 确保扫描开关是开启的，

@Configuration
@ComponentScan("com.demo.ioc")
public class SpringConfiguration {

}

② 在userServiceImpl类上面加@Component注解就行了

@Component
public class UserServiceImpl implements UserService{
    @Override
    public void create() {

    }

    @Override
    public void delete() {

    }
}

# 方式二：配置类注册

在配置类中使用@Bean标签注册组件

@Configuration
public class SpringConfiguration {
	@Bean
    public UserService userService() {
        UserService userService = new UserServiceImpl();
        return userService;
    }
}

# 基于配置类

//按类名获取

UserService userService3 = applicationContext.getBean("userService", UserService.class);

组件id：

在@Bean注册方式中，组件id默认是方法名，可以通过给@Bean注解的value赋值来设置id

@Bean("userService")

在类直接注册方式中，组件id默认是类名首字母小写，也可以通过给组件注解的value赋值来设置id

@Component("userService")

# 依赖注入(DI)

依赖注入就是给容器中的对象的属性赋值，完成相应的配置后，一般可以由Spring自动完成

# 代理

通过代理处理数据操作的事务

public class ProxyUtil {
	//t 就是要代理的实例
    public static <T> T getServiceProxy(T t){
        Class<?> clazz = t.getClass();
        T proxy = (T) Enhancer.create(clazz, new InvocationHandler() {
            @Override
            public Object invoke(Object o, Method method, Object[] objects) throws Throwable {
                //获得该类的实例
                SqlSession sqlSession = MyBatisUtil.getSqlSession();
                //通过反射获取该实例的所有成员变量
                Field[] declaredFields = clazz.getDeclaredFields();
				//遍历实例的成员变量，看是否有以名字Mapper结尾的变量，有的话，就通过sqlsession为他们获取一个对应的Mapper对象
                for (Field declaredField : declaredFields) {
                    if(declaredField.getName().endsWith("Mapper")) {
                        declaredField.setAccessible(true);
                        //getType():获取该变量的类型，使用getClass()是获取的Field的类对象
                        Object mapper = sqlSession.getMapper(declaredField.getType());
                        //为该变量赋值 set(要赋值的对象, 赋的值)
                        declaredField.set(t, mapper);
                    }
                }
                // 通过反射执行Service中的方法
                Object invoke = method.invoke(t, objects);
                //提交事务，关闭SqlSession
                sqlSession.commit();
                sqlSession.close();
                return invoke;
            }
        });
        return proxy;
    }
}

# FactoryBean

FactoryBean用于注册特定的一个组件

而BeanFactory用于注册所有的组件，

使用方法：

public class UserServiceFactoryBean implements FactoryBean<UserService> {
    @Autowired
    @Qualifier("userServiceImpl")
    UserService userService;
    @Override
    public UserService getObject() throws Exception {
        return ProxyUtil.getServiceProxy(userService);
    }

    @Override
    public Class<?> getObjectType() {
        return UserService.class;
    }
}

注册FactoryBean组件

① 直接加注解@Component

② @Bean注册组件

方法②比较常用，因为通常是第三方提供的类，已经编译了，只能用第二种方法

@Configuration
@ComponentScan("com.demo")
public class AppConfiguration {
    @Bean
    public UserServiceFactoryBean userServiceFactoryBean() {
        return new UserServiceFactoryBean();
    }
}

使用FactoryBean

@RunWith(SpringJUnit4ClassRunner.class)

        user.setPassword("123456");

    }
}
